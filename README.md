# Каталог сувенирной продукции

Next.js MVP для публичного каталога и отдельной админки. Интерфейс полностью на русском языке.

## Архитектура

- Публичный каталог `/` читает опубликованную версию из `data/catalog.published.json`.
- Админка `/admin` работает с черновиком `data/catalog.draft.json`.
- При нажатии «Сохранить черновик» изменения сохраняются в браузере.
- При нажатии «Опубликовать» draft копируется в published в браузере. Публичная страница начинает показывать опубликованную локальную версию.
- Для бесплатного production-сценария выбран GitHub repository storage: JSON-файлы в `data`, изображения в `public/products`, сохранение через GitHub API.
- Owner вводит GitHub token в админке. Token хранится только в localStorage браузера и не попадает в репозиторий.

Такой вариант подходит для бесплатного размещения на GitHub Pages и не требует базы данных. Для production с повышенными требованиями к безопасности лучше перейти на Firebase Auth или Supabase Auth, потому что хранение пользователей в JSON даже с PBKDF2-хэшами уступает полноценному auth-сервису.

## Запуск локально

```bash
npm install
npm run dev
```

Откройте:

- публичный каталог: `http://localhost:3000`
- админка: `http://localhost:3000/admin`

## Первый вход в админку

1. Откройте `/admin`.
2. Если пользователей ещё нет, форма создаст первого пользователя.
3. Первый пользователь автоматически получает роль `owner`.
4. Owner может открыть «Управление доступом» и создать сотрудников с ролью `editor`.
5. Editor видит только «Настройку каталога».

## Как добавить товар

1. Войдите в `/admin`.
2. В разделе «Настройка каталога» нажмите «Добавить».
3. Заполните обязательные поля со звёздочкой.
4. Загрузите PNG-изображение.
5. Проверьте предпросмотр справа.
6. Нажмите «Сохранить черновик».
7. Когда всё готово, нажмите «Опубликовать».

Публичный каталог не показывает изменения черновика до публикации.

## PDF

На публичной странице пользователь отмечает товары галочками и нажимает «Скачать PDF». Если ничего не выбрано, интерфейс показывает предупреждение. PDF формируется только из выбранных позиций, группируется по разделам и не разрывает карточки между страницами.

## GitHub token

Для сохранения JSON в репозиторий через админку нужен Personal Access Token.

Минимальный вариант:

1. GitHub → Settings → Developer settings → Personal access tokens.
2. Создайте fine-grained token для нужного репозитория.
3. Разрешите `Contents: Read and write`.
4. В админке заполните:
   - владелец репозитория;
   - имя репозитория;
   - ветка, обычно `main`;
   - token.
5. Нажмите «Сохранить в GitHub».

Изображения, загруженные в текущем MVP, сразу работают в предпросмотре через data URL. Для production-доработки следующий шаг: сохранять PNG через GitHub API в `public/products` и подставлять путь `/products/<file>.png`.

## Размещение на GitHub Pages

1. Создайте репозиторий на GitHub.
2. Запушьте проект.
3. В настройках репозитория включите Pages через GitHub Actions.
4. Workflow `.github/workflows/deploy.yml` соберёт статическую версию Next.js в `out`.

```bash
git init
git add .
git commit -m "Создать MVP каталога сувенирной продукции"
git branch -M main
git remote add origin https://github.com/<owner>/<repo>.git
git push -u origin main
```

## Основные файлы

- `app/page.tsx` — публичный каталог.
- `app/admin/page.tsx` — админка.
- `components/ProductCard.tsx` — карточка товара.
- `components/PdfExport.tsx` — PDF-выгрузка.
- `lib/types.ts` — типы `User`, `Role`, `Product`, `Category`, `CorporateColor`, `ClothingSize`, `Material`, `CardBackgroundColor`, `CatalogData`.
- `data/catalog.draft.json` — черновик.
- `data/catalog.published.json` — опубликованный каталог.
- `data/users.json` — пользователи.
