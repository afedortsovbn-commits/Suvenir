# Каталог сувенирной продукции

Next.js MVP для публичного каталога и отдельной админки. Интерфейс полностью на русском языке.

## Архитектура

- Публичный каталог `/` читает опубликованную версию из GitHub: `data/catalog.published.json`.
- Админка `/admin` читает черновик, опубликованную версию и пользователей из репозитория `afedortsovbn-commits/Suvenir`.
- При нажатии «Сохранить черновик» админка коммитит `data/catalog.draft.json` в GitHub.
- При нажатии «Опубликовать» админка коммитит `data/catalog.draft.json` и `data/catalog.published.json` в GitHub.
- Пользователи сохраняются в `data/users.json`.
- Owner вводит GitHub token в админке. Token нужен только для записи и хранится в памяти открытой страницы.

Такой вариант работает с любого компьютера: источник данных находится в GitHub, а GitHub Pages показывает последнюю опубликованную версию после сборки. Для production с повышенными требованиями к безопасности лучше перейти на Firebase Auth или Supabase Auth, потому что хранение пользователей в JSON даже с PBKDF2-хэшами уступает полноценному auth-сервису.

## Запуск локально

```bash
npm install
npm run dev
```

Откройте:

- публичный каталог: `http://localhost:3000`
- админка: `http://localhost:3000/admin`

## Первый вход в админку

1. Откройте `/admin`.
2. Введите GitHub token с правом `Contents: Read and write`.
3. Если пользователей ещё нет, форма создаст первого пользователя и сохранит его в `data/users.json`.
4. Первый пользователь автоматически получает роль `owner`.
5. Owner может открыть «Управление доступом» и создать сотрудников с ролью `editor`.
6. Editor видит только «Настройку каталога».

## Как добавить товар

1. Войдите в `/admin`.
2. В разделе «Настройка каталога» нажмите «Добавить».
3. Заполните обязательные поля со звёздочкой.
4. Загрузите PNG-изображение.
5. Проверьте предпросмотр во вкладке «Превью».
6. Нажмите «Сохранить позицию».
7. Нажмите «Сохранить черновик», чтобы записать изменения в GitHub.
8. Когда всё готово, нажмите «Опубликовать».

Публичный каталог не показывает изменения черновика до публикации.

## PDF

На публичной странице пользователь отмечает товары галочками и нажимает «Скачать PDF». Если ничего не выбрано, интерфейс показывает предупреждение. PDF формируется только из выбранных позиций, группируется по разделам и не разрывает карточки между страницами.

## GitHub token

Для сохранения JSON в репозиторий через админку нужен Personal Access Token.

Минимальный вариант:

1. GitHub → Settings → Developer settings → Personal access tokens.
2. Создайте fine-grained token для нужного репозитория.
3. Разрешите `Contents: Read and write`.
4. В админке заполните:
   - владелец репозитория;
   - имя репозитория;
   - ветка, обычно `main`;
   - token.
5. Нажмите «Сохранить в GitHub».

Изображения из мокового каталога лежат в `public/products`. Следующий production-шаг для загрузки новых изображений: сохранять PNG через GitHub API в `public/products` и подставлять путь `/products/<file>.png`.

## Размещение на GitHub Pages

1. Создайте репозиторий на GitHub.
2. Запушьте проект.
3. В настройках репозитория включите Pages через GitHub Actions.
4. Workflow `.github/workflows/deploy.yml` соберёт статическую версию Next.js в `out`.

```bash
git init
git add .
git commit -m "Создать MVP каталога сувенирной продукции"
git branch -M main
git remote add origin https://github.com/<owner>/<repo>.git
git push -u origin main
```

## Основные файлы

- `app/page.tsx` — публичный каталог.
- `app/admin/page.tsx` — админка.
- `components/ProductCard.tsx` — карточка товара.
- `components/PdfExport.tsx` — PDF-выгрузка.
- `lib/types.ts` — типы `User`, `Role`, `Product`, `Category`, `CorporateColor`, `ClothingSize`, `Material`, `CardBackgroundColor`, `CatalogData`.
- `data/catalog.draft.json` — черновик.
- `data/catalog.published.json` — опубликованный каталог.
- `data/users.json` — пользователи.
